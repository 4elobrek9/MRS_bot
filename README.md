# MRS_bot — Telegram RPG + AI bot

MRS_bot — это Telegram-бот на `aiogram 3` с RPG-экономикой, профилями, RP-действиями, квестами, казино, промокодами и AI-режимами для общения.

## Что исправлено (важно)

В проекте исправлены проблемы, которые часто приводили к ситуации «бот запустился, но не отвечает»:

1. **Единый `Dispatcher` для всех хендлеров.**
   Ранее часть хендлеров (`/start`, `/mode`, `/stats`, `/joke`, приватный чат) регистрировалась в одном экземпляре `Dispatcher`, а polling запускался на другом. Из-за этого бот мог «молчать» в ЛС или не видеть часть команд.
2. **Исправлена инициализация promo-хендлеров.**
   Передавались аргументы в неверном порядке.
3. **Ускорен fallback для Mistral.**
   `bot.get_me()` теперь с timeout, чтобы бот не «висел» долго на старте при сетевых проблемах.
4. **Исправлен SQL-typo в миграции инвентаря** (`DEFAULT 1`).

## Быстрый старт

### 1) Требования

- Python 3.10+
- Telegram bot token от `@BotFather`

### 2) Установка

```bash
git clone https://github.com/4elobrek9/MRS_bot
cd MRS_bot
python -m venv .venv
source .venv/bin/activate      # Linux/macOS
# .venv\Scripts\activate      # Windows
pip install -r requirements.txt
```

### 3) Переменные окружения

Создайте `.env` в корне проекта:

```env
TOKEN=YOUR_TELEGRAM_BOT_TOKEN
OLLAMA_API_BASE_URL=http://localhost:11434
OLLAMA_MODEL_NAME=llama3
ADMIN_USER_ID=
CHANNEL_ID=
MISTRAL_API_KEY=
```

> `MISTRAL_API_KEY` опционален. Если ключа нет или сеть недоступна — бот все равно работает, просто без Mistral group-chat функционала.

### 4) Запуск

```bash
python main.py
```

## Почему бот может не отвечать: диагностика

### 1. Проверьте, что polling действительно стартовал

В логах должно быть:

- `Запуск поллинга...`

Если этой строки нет — инициализация упала раньше.

### 2. Проверьте токен

- Неверный `TOKEN` часто даёт `Unauthorized`/`TelegramBadRequest`.
- Убедитесь, что запускаете бота именно с тем `.env`, который ожидаете.

### 3. Проверьте privacy mode для групп

Для реакций на обычный текст в группе у бота должен быть корректный режим privacy у `@BotFather`.

### 4. Проверьте конфликт вебхука

Если раньше использовался webhook, polling может не получать апдейты.

### 5. Проверьте зависимости AI-модулей

- `Ollama` нужен для приватного AI-чата с локальной моделью.
- При недоступном Ollama часть AI-ответов может падать, но базовые команды должны работать.

## Основные команды

### Общие
- `/start` — старт и приветствие
- `/help` — справка
- `/mode` — выбор режима общения
- `/stats` — статистика
- `/joke` — анекдот

### RPG / Экономика
- `/profile` — профиль
- `/inventory` — инвентарь
- `/work` — работа
- `/shop` — магазин
- `/pshop` — PLUM-магазин
- `/quests` — задания
- `/top` — топ игроков
- `/give <sum> @username` — перевод LUM
- `/market`, `/auction`, `/invest` — торговля/аукцион/инвестиции

### RP
- `/myhp` (и текстовые аналоги) — HP
- `/heal` — лечение
- `/rpactions` — список RP-действий

## Структура проекта (главное)

- `main.py` — точка входа, инициализация БД/роутеров/polling
- `core/main/ez_main.py` — базовая конфигурация, bot/dp/logging/env
- `core/main/dec_command.py` — приватные AI/utility-команды
- `group_stat.py` — профиль, экономика, квесты, выдача валюты
- `rp_module_refactored.py` — RP-модуль
- `core/group/RPG/` — RPG механики
- `core/group/casino.py` — казино
- `core/group/promo.py` — промокоды
- `database.py` — общая БД
